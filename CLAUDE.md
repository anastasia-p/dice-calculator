# DICE-калькулятор | Quality Trek

Telegram Mini App для оценки вероятности успешного внедрения изменений по методологии BCG (Boston Consulting Group).

Автор: Анастасия Пономарева, Quality Trek (quality-trek.ru)

---

## Суть продукта

DICE — фреймворк BCG для оценки инициатив по 4 параметрам:
- **D** (Duration) — частота пересмотра прогресса, коэффициент ×1
- **I** (Integrity) — компетентность команды, коэффициент ×2
- **C1** (Commitment) — поддержка руководства, коэффициент ×2
- **C2** (Commitment) — вовлеченность команды, коэффициент ×1
- **E** (Effort) — нагрузка сверх основной работы, коэффициент ×1

**Формула:** `D + 2×I + 2×C1 + C2 + E`
- Выигрыш: 7–13 (зеленый #639922)
- Беспокойство: 14–17 (желтый #BA7517)
- Проблема: 18+ (красный #E24B4A)

Каждый параметр оценивается по шкале 1–4. Минимальный балл = 7, максимальный = 28.

## Экраны

### Экран 1 — Старт (start.js)
Две карточки выбора режима:
1. **Оценю сам** — переход сразу на calculator.js (одиночный режим, без Firebase)
2. **Работа в команде** — переход на setup.js

### Экран 2 — Настройка сессии (setup.js)
Только для организатора командного режима.

Поля:
- Название проекта (обязательное)
- Параметры оценки — список D/I/C1/C2/E с кнопкой ✎ для редактирования вариантов ответов

После нажатия "Создать сессию":
- Кнопка меняется на `#41bfd0` "Сессия создана" на 1 секунду
- Потом появляется бирюзовая плашка-подсказка: "Отправьте ссылку всем участникам — каждый заполнит оценку независимо, и вы увидите расхождения"
- Ниже две кнопки с подсказками:
  - "⬆ Поделиться" / подсказка: "Выберите куда отправить — шторка с приложениями"
  - "⎘ Скопировать" / подсказка: "Ссылка скопируется, апп свернется"
- Разделитель
- "→ Перейти к заполнению" / подсказка: "Заполните свою оценку как организатор"

Шаринг реализован через `navigator.share()` (нативная шторка iOS/Android), фоллбек — `navigator.clipboard`. Сворачивание — `Telegram.WebApp.minimize()`.

### Экран 3 — Калькулятор (calculator.js)
Единый для одиночного и командного режима.

**В командном режиме** вверху добавляется карточка с названием проекта (только чтение) и полем "Ваше имя" (необязательное, дефолт "Участник N").

**Структура:**
- 5 карточек параметров (D, I, C1, C2, E)
- У каждой: буква + название + кнопка ? (подсказка)
- В одиночном режиме также кнопка ✎ (редактирование вариантов)
- 4 варианта ответа с radio-dot и цифрой
- Блок результата (появляется только когда все 5 заполнены):
  - Прогресс-бар с точками
  - Формула с чипами
  - Счет + зона
  - Полоска прогресса
  - Описание зоны
  - Кнопка "⎘ Скопировать результат" (одиночный) или "✓ Отправить оценку" (командный)

**Цвет поля "Название проекта" при фокусе** и **кнопки при наведении** меняется в цвет текущей зоны (зеленый/желтый/красный).

**Кнопка "Отправить оценку"** после нажатия: `#41bfd0`, текст "Оценка отправлена", переход на waiting.js.

### Экран 4 — Ожидание (waiting.js)
Показывается после отправки оценки в командном режиме.

- Карточка с названием проекта
- Прогресс-бар (ответили X из Y)
- Список участников с аватарками (первая буква + уникальный цвет), статус ✓/ждем
- Бирюзовая плашка: "Результаты появятся когда организатор завершит сессию"
- 6px отступ
- **Только для организатора:**
  - "⬆ Поделиться ссылкой ещё раз"
  - Разделитель
  - "⏹ Завершить сессию" / подсказка: "Результаты станут доступны всем участникам"

Данные обновляются в реальном времени через Firebase.

### Экран 5 — Результаты (results.js)
Показывается всем после завершения сессии организатором.

**Блок итоговых баллов:**
- Каждый участник: цветная аватарка + имя + балл + зона (Выигрыш/Беспокойство/Проблема)
- Среднее значение и зона

**Блок расхождений по параметрам:**
- Подзаголовок: "! — стоит обсудить с командой"
- Легенда: зеленая точка (единодушно), желтая (расхождение), красная (критическое)
- По каждому параметру: буква + название + плашка статуса + кнопка ?
- Чипы с оценками: цветная аватарка участника + его цифра

**Логика плашек:**
- "единодушно" (зеленая) — все выбрали одинаково
- "! расхождение" (желтая) — есть разброс
- "! критическое" (красная) — одновременно присутствуют оценки 1 и 4

**Цвета участников** (назначаются по порядку входа через `colorIndex % COLORS.length`):
```js
const PARTICIPANT_COLORS = [
  { bg: 'rgba(66,190,207,0.2)',   text: '#0a7a8a' }, // бирюзовый
  { bg: 'rgba(139,92,246,0.15)',  text: '#6d28d9' }, // фиолетовый
  { bg: 'rgba(249,115,22,0.15)',  text: '#c2410c' }, // оранжевый
  { bg: 'rgba(16,185,129,0.15)',  text: '#065f46' }, // зеленый
  { bg: 'rgba(236,72,153,0.15)',  text: '#9d174d' }, // розовый
  { bg: 'rgba(234,179,8,0.15)',   text: '#854d0e' }, // желтый
  { bg: 'rgba(99,102,241,0.15)',  text: '#3730a3' }, // индиго
  { bg: 'rgba(239,68,68,0.15)',   text: '#991b1b' }, // красный
];
// Назначение: colorIndex = порядковый номер входа в сессию
// Цвет = PARTICIPANT_COLORS[colorIndex % PARTICIPANT_COLORS.length]
// Таким образом поддерживается любое количество участников
```

Кнопка "⎘ Скопировать результаты" внизу.

## Telegram боты

Стек: Python, aiogram 3, firebase-admin.

### Боты
- **Продакшн:** `@DICE_bcg_bot`
- **Тест:** создать через @BotFather, например `@DICE_bcg_test_bot`

Бот нужен для:
1. Создания сессий через команду `/new`
2. Отправки уведомления в чат когда организатор завершил сессию (опционально)

Ссылка на сессию: `t.me/DICE_bcg_bot?startapp=SESSION_ID`

При открытии мини-аппа по ссылке — `SESSION_ID` передается через `Telegram.WebApp.initDataUnsafe.start_param`.

## Среды разработки (Dev / Prod)

### Ветки репозитория
- `main` — продакшн, деплоится на GitHub Pages автоматически
- `dev` — разработка и тесты, деплоится на отдельный URL или тестируется локально

### Два Firebase проекта
- `dice-calculator-prod` — продакшн
- `dice-calculator-test` — тестовая среда

### Переменные окружения (.env, не коммитить в гит!)
```
# Тестовая среда
BOT_TOKEN=токен_тестового_бота
FIREBASE_URL=https://dice-calculator-test.firebaseio.com
ENVIRONMENT=test

# Продакшн (на Railway)
BOT_TOKEN=токен_продакшн_бота
FIREBASE_URL=https://dice-calculator-prod.firebaseio.com
ENVIRONMENT=production
```

Файл `.env` добавить в `.gitignore`. Для фронтенда Firebase конфиг переключается через `js/config.js` который читает `ENVIRONMENT`.

## Важные UX-решения

1. **Результат не показывается** пока не заполнены все 5 параметров
2. **Имя участника необязательно** — дефолт "Участник N" по порядку входа
3. **Редактировать варианты ответов** может только организатор (на экране 2), участники видят уже отредактированные варианты
4. **Завершить сессию** может только организатор (кнопка на экране 4)
5. **Переоценки нет** — слишком сложная логика для MVP
6. **Шаринг** через `navigator.share()`, сворачивание через `Telegram.WebApp.minimize()`
